# C Programming Study Guide - Part 5

## 6.17: Separate Files

### Overview
Separating part of a program’s code into separate files improves manageability and reusability.

### Benefits:
- Prevents the main file from becoming too large.
- Allows functions to be reused in other programs.

### Example:
We split a program into:
- `main.c` (contains the main function)
- `threeintsfcts.c` (contains definitions of related functions)
- `threeintsfcts.h` (contains function declarations)

#### Code Breakdown:

**main.c**
```c
[[include]] <stdio.h>
[[include]] "threeintsfcts.h"

int main(void) {
    printf("%d\n", ThreeIntsSum(5, 10, 20));
    printf("%d\n", ThreeIntsAvg(5, 10, 20));
    return 0;
}
```

**threeintsfcts.c**
```c
int ThreeIntsSum(int num1, int num2, int num3) {
    return (num1 + num2 + num3);
}

int ThreeIntsAvg(int num1, int num2, int num3) {
    int sum = num1 + num2 + num3;
    return (sum / 3);
}
```

**threeintsfcts.h**
```c
int ThreeIntsSum(int num1, int num2, int num3);
int ThreeIntsAvg(int num1, int num2, int num3);
```

### Compilation
To compile:
```sh
gcc -Wall main.c threeintsfcts.c
```

### Header File Guards
To prevent multiple inclusions:
```c
[[ifndef]] THREEINTSFCTS_H
[[define]] THREEINTSFCTS_H

int ThreeIntsSum(int num1, int num2, int num3);
int ThreeIntsAvg(int num1, int num2, int num3);

[[endif]]
```

### Quiz Highlights:
1. **False** – Header files do not have to end with `.h`.
2. **False** – Header files should **declare**, not define, functions.
3. **True** – Guards prevent multiple inclusion.
4. **False** – Correct format: `#ifndef`, not `#ifdef`.
